== Preface

Software engineering organizations are increasingly instrumented, yet they often remain poorly understood. They can report pull-request counts, deployment frequency, incident volume, and tool usage, but still struggle to explain where expert attention is spent, which process changes create value, or whether an AI-assisted workflow genuinely improves engineering outcomes.

_Engineering Intelligence_ develops a framework for answering those questions. It treats engineering as a measurable socio-technical system and integrates engineering economics, human cognition, software architecture, queueing theory, causal inference, and AI-assisted development.
